# Зачем оно надо?
- Смысл приложения - используя API Teyca создать приложение, которое будет держать синхронную базу клиентов для использования сторонних "фич".
- Потыкать в FastAPI.
## Проблематика
- Огрничение со стороны вендора (Teyca) на количество запросов;
- Недостаток функционала для реализации программ от отдела маркетинга;
- Тяжелая аналитика в размахе общей базы по позициям;
- Ограниченная информация передающаяся с кассы;
## Решение проблемы
- Использоавть webhook для сбора информации из системы;
- Обрабатывать кастомную логику у себя  и высылать ответы только при соблюдении всех условий (скользящее окно и ограничение на количество запросов)
- Подключение к кассовой бд (из ETL Frontol) и синхронизация операций для подробной аналитики.
